Module 2: React Overview

What is React

  • A fast and declarative UI library used at Facebook, Instagram, Netflix, AirBnb, and many others
  • Runs on multiple targets: DOM, canvas, native, terminal, etc
  • Uses a virtual DOM unidirectional data-binding system to only update the parts of the DOM that need to change
  • A way to bring functional programming to building user interfaces

Why React

  • Forces you to think in terms of small components
  • Clear seperation between state and static properties
  • Extremely simple yet powerful API
  • Easy to debug
  • Fast and low memory footprint
  • Works well with third-party libraries
  • Works well with existing codebases
  • Battle tested. With just Netflix and Facebook alone that's 30% of the internet
  • It's used everywhere now. It's the new jQuery

Thinking in React

  • MVCs make sense on the server, not the browser
  • UIs are trees/hierchies of components
  • You should architect your UIs are components within components withing components
  • It's easier to reason about state if it lives in one place
  • It's easier to reason about data if it only flows from parent to child
