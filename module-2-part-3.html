Reusing Components

When you think about it, the DOM is a tree of DOM nodes. React works the same way. Every component has a parent, other than the root component. A React application is a tree of React components.

This fact allows React to be very fast. It keeps an in-memory representation of the DOM as a tree. Whenever React needs to update the DOM, it will create a new in memory representation of the DOM and diff the new tree with the old tree. That way React knows the minimum number of DOM mutations necessary to update the DOM to the new state of the UI.

This means a React component must return a tree structure as well. Your React components must have just one return value. That is why in the next example the Header component must wrap its rendered elements in a div.
