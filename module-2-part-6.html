Props vs State

So far you've learned about props and state. How do you know when to use state or props? State is for data that you know will change due to some external input. Props can be thought of as static "properties".

To properly separate concerns, aim to create small stateless child components that act like functions in that they take a set of arguments (props) and always returns the same value based on that input. Keep shared mutable state in a parent component that also contains the business logic.
